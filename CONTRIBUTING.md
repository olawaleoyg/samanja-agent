# Contributing

Thanks for considering a contribution!

## Commit message convention

This repo uses [Conventional Commits](https://www.conventionalcommits.org/)
so that `release-please` can automatically determine version bumps and
generate the changelog. Use prefixes like:

- `feat: add health-check step to deploy pipeline` -> minor version bump
- `fix: correct path sandbox edge case` -> patch version bump
- `chore: update dependencies` -> no version bump, included in changelog
- `feat!: change tool schema` or `BREAKING CHANGE:` in the body -> major bump

`CHANGELOG.md` is now generated by release-please from these commit
messages via the standing release PR -- avoid hand-editing it directly
once release-please is active.
